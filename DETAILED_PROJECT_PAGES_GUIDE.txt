==================================================
GLOBAL APP STRUCTURE (SHARED BEHAVIOR)
==================================================

Most authenticated pages load these shared scripts:
- js/api.js: API client, token handling, auto refresh of access token.
- js/utils.js: helpers for currency/date formatting, toasts, charts, icons, modals.
- js/app.js: sidebar behavior, user info in shell, theme toggle, logout, chatbot widget, notification bubbles.

Authentication model:
- JWT tokens are stored in local storage.
- Access token key: sf_token
- Refresh token key: sf_refresh_token
- Cached user key: sf_user
- When a request gets 401, api.js attempts POST /api/auth/refresh/ and retries once.
- If refresh fails, user is redirected to login.html.

Theme and shell model:
- Pages with sidebar have menu toggle, user header, theme toggle, logout button.
- Theme is switched on root document attribute and reflected in charts/maps.

Key IDs:
- loginForm, email, password, loginBtn, loginSpinner, authError, googleLoginBtn

Detailed workflow:
1. User enters email/password and submits login form.
2. auth.js calls API.login(email, password) -> POST /api/auth/login/.
3. On success, token and user are cached.
4. auth.js calls API.getProfile() -> GET /api/auth/profile/ to route correctly.
5. Redirect target is based on role and onboarding state.
6. Google flow uses API.googleLogin(token) -> POST /api/auth/google-login/.

Error and edge handling:
- Client-side empty-field checks.
- Server/API errors displayed in authError.
- If backend cannot be reached, user gets a clear connectivity hint.

   API.getAdminUsers(params) -> GET /api/auth/admin/users/.
3. Update selected user role/plan flags:
   API.updateAdminUser(userId, payload) -> PATCH /api/auth/admin/users/{id}/.
4. Load audit trail:
   API.getAdminAuditLogs(params) -> GET /api/auth/admin/audit-logs/.

Error and edge handling:
- Separate loading/reload paths for users vs logs.
- Defensive rendering when lists are empty.
- Toast feedback for update failures.
